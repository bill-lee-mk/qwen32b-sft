# 提示词存放位置与迁移指南

## 独立提示词包（推荐分享）

**位置**：`evaluation_output/prompt_bundle.json`（运行 `python scripts/export_standalone_prompt.py` 生成）

**分享给他人**：提供此文件 + `scripts/run_with_bundle.py`，对方只需 `pip install openai` 和 API Key 即可生成，无需本仓库其他代码。

```bash
# 对方操作
pip install openai
export DEEPSEEK_API_KEY=sk-xxx
python run_with_bundle.py --bundle prompt_bundle.json --all --model deepseek-reasoner --output mcqs.json
```

**能否达到最高分**：能。bundle 中的 prompt 与 `generate_mcq.py` 使用的完全一致（同一套 examples、prompt_rules、build_full_prompt 逻辑），预期通过率与闭环最佳结果一致（如 96.6%）。`run_with_bundle.py` 的 JSON 解析较简化，若遇解析失败可重试或改用完整仓库的 `generate_mcq.py`。

### 使用 bundle 生成题目并评分（完整流程）

若已导出 `prompt_bundle_deepseek-reasoner.json` 和 `prompt_bundle_kimi-k2.5.json`：

**1. 生成题目**

```bash
# DeepSeek-reasoner
export DEEPSEEK_API_KEY=sk-xxx
python scripts/run_with_bundle.py \
  --bundle evaluation_output/prompt_bundle_deepseek-reasoner.json \
  --all --model deepseek-reasoner \
  --output evaluation_output/mcqs_237_deepseek-reasoner.json

# Kimi-k2.5（默认 15 并发；遇 429 可 --workers 5 或 1）
export KIMI_API_KEY=xxx
python scripts/run_with_bundle.py \
  --bundle evaluation_output/prompt_bundle_kimi-k2.5.json \
  --all --model kimi-k2.5 \
  --output evaluation_output/mcqs_237_kimi-k2_5.json
```

**2. 评估/评分**（需 InceptBench token，在**本仓库**中运行）

```bash
export INCEPTBENCH_API_KEY=xxx   # 或 INCEPTBENCH_TOKEN / EVALUATOR_TOKEN
python main.py evaluate \
  --input evaluation_output/mcqs_237_deepseek-reasoner.json \
  --output evaluation_output/results_237_deepseek-reasoner.json \
  --parallel 20
```

评估结束后会打印通过率（≥0.85 视为通过），结果 JSON 含每题的 `overall_score` 和 `scores`。

## 一、提示词存放位置（完整仓库）

| 文件/目录 | 路径 | 说明 |
|-----------|------|------|
| **Few-shot 示例** | `processed_training_data/examples.json` | 按 (standard, difficulty) 分组的高分示例，供模型模仿 |
| **动态规则** | `processed_training_data/prompt_rules.json` | 从失败反馈提取的全局/针对性规则 |
| **标准描述** | `data/standard_descriptions.json` | CCSS 各标准的文字描述，用于 user prompt |
| **System 模板** | `data_processing/build_prompt.py` | 角色、约束、规则列表等，写在代码中 |

多模型/多批次时会有 `examples_{model}_{run_id}.json`、`prompt_rules_{model}_{run_id}.json`，用法相同。

## 二、在新机器上使用微调好的提示词

### 2.1 需要复制的文件

```
# 提示词相关（必需）
processed_training_data/examples.json          # 或 examples_deepseek-reasoner_0213_2.json 等
processed_training_data/prompt_rules.json     # 或 prompt_rules_deepseek-reasoner_0213_2.json 等
data/standard_descriptions.json

# 代码（必需，用于构建 prompt 和调用 API）
data_processing/build_prompt.py
data_processing/select_examples.py
data_processing/analyze_dimensions.py
scripts/generate_mcq.py
main.py
```

### 2.2 最小化部署（仅生成，不训练）

1. **克隆或复制项目**（至少包含上述文件及目录结构）

2. **安装依赖**（仅 API 生成所需）：
   ```bash
   pip install openai  # DeepSeek、Kimi、GPT 等
   pip install google-generativeai  # 若用 Gemini
   ```

3. **设置 API Key**：
   ```bash
   export DEEPSEEK_API_KEY="sk-xxx"   # DeepSeek
   export KIMI_API_KEY="xxx"          # Kimi
   # 或运行时 --api-key
   ```

4. **指定示例与规则**（若路径不同）：
   ```bash
   python scripts/generate_mcq.py \
     --model deepseek-reasoner \
     --examples /path/to/examples.json \
     --all-combinations \
     --output mcqs_237.json
   ```

   `prompt_rules` 默认从 `processed_training_data/prompt_rules.json` 读取；若路径不同，可设置环境变量：
   ```bash
   export PROMPT_RULES_PATH=/path/to/prompt_rules.json
   python scripts/generate_mcq.py --model deepseek-reasoner --all-combinations
   ```

### 2.3 一键打包迁移

在**原机器**上打包提示词与必要代码：

```bash
# 创建迁移包
mkdir prompt_migration_pkg
cp processed_training_data/examples.json processed_training_data/prompt_rules.json prompt_migration_pkg/
mkdir -p prompt_migration_pkg/data
cp data/standard_descriptions.json prompt_migration_pkg/data/
cp -r data_processing scripts main.py prompt_migration_pkg/
tar -czvf prompt_migration_pkg.tar.gz prompt_migration_pkg/
```

在**新机器**上：

```bash
tar -xzvf prompt_migration_pkg.tar.gz
cd prompt_migration_pkg
pip install openai
export DEEPSEEK_API_KEY="sk-xxx"
python scripts/generate_mcq.py --model deepseek-reasoner --all-combinations --output mcqs.json
```

## 三、路径说明

- `--examples`：指定 few-shot 示例 JSON，默认 `processed_training_data/examples.json`
- `PROMPT_RULES_PATH`：指定动态规则 JSON，默认 `processed_training_data/prompt_rules.json`
- `data/standard_descriptions.json`：需与 `build_prompt.py` 同项目下，或修改 `_STANDARD_DESCRIPTIONS_PATH`
