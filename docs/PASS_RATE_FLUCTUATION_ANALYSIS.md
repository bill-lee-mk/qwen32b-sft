# 闭环通过率波动分析

## 现象

从 `log_237_deepseek-reasoner.log` 可见，通过率 (>=0.85) 随轮次变化：

| 轮次 | 通过率 | 通过/有效 |
|------|--------|-----------|
| R1 | 70.5% | 167/237 |
| R2 | 86.1% | 204/237 |
| R3 | 93.7% | 222/237 |
| R4 | 94.5% | 224/237 |
| R5 | 92.0% | 218/237 |
| R6 | **64.1%** | 152/237 |
| R7 | 90.3% | 214/237 |
| R8 | 92.4% | 219/237 |
| R9 | **96.6%** | 229/237 |
| R10 | 93.2% | 221/237 |

**趋势**：先升（R1→R4）→ 下降（R5、R6）→ 再升（R7→R9）→ 略有回落（R10）。

## 原因分析

### 1. 生成具有随机性

- `generate_mcq.py` 中 DeepSeek 使用 `temperature=0.7`，相同 prompt 会得到不同输出。
- 每轮都会**重新生成** 237 题，不是复用上一轮题目。
- 因此，即便 prompt 规则相同，每轮题目质量也会有波动。

### 2. Prompt 规则每轮动态变化

- 每轮结束后会执行 `improve-examples` 和 `improve-prompt`。
- 规则和示例会随轮次更新，可能带来：
  - 有效改进（通过率提升）
  - 过拟合或引入噪声（通过率下降，如 R6）

### 3. 评估分数边界效应

- 0.82–0.84 的题目接近 0.85 阈值，评分本身可能有小幅波动。
- 若干题目在边界附近，会导致通过率在几轮间上下浮动。

### 4. 结论

**通过率波动是预期现象，不是实现错误。** 每次运行的结果确实存在不确定性，这是由 LLM 随机性、prompt 迭代和评估边界共同造成的。建议关注「历史最高通过率」和「最佳轮次」作为更稳定的指标。
